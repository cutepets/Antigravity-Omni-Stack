# Antigravity Start Here

Entry point for the active Antigravity workflow in this repository.

This file is intentionally short. Do not treat old counts, marketing copy, or legacy examples as runtime truth.

## Read Order

1. `.agent/rules/GEMINI.md`
Reason: core constitution, routing expectations, safety posture.

2. `.agent/docs/ANTIGRAVITY_ROUTING.md`
Reason: active routing by work size, specialization, and execution mode.

3. `.agent/docs/AGENT_STANDARD.md`
Reason: canonical structure for specialist agents, handoff, deliverables, and definition of done.

4. `.agent/docs/CODEX_COLLABORATION.md`
Reason: operating model when Codex works alongside Antigravity.

## Operating Principle

- Antigravity is the framework and governance layer.
- The user is the real dispatcher.
- Agents are specialist playbooks, not autonomous coworkers unless the runtime explicitly supports that behavior.
- GitNexus is the primary code intelligence layer for understanding code, impact, and execution flow.

## Work Modes

- Quick work: direct execution with the correct specialist mindset.
- Standard work: lightweight planning plus targeted execution.
- Heavy work: research, planning, structured handoff, staged execution, verification.

See `.agent/docs/ANTIGRAVITY_ROUTING.md` for the full matrix.

## Canonical Runtime Areas

- `.agent/rules/` -> governance and safety
- `.agent/agents/` -> specialist role playbooks
- `.agent/skills/` -> reusable deep methods
- `.agent/workflows/` and `.agent/get-shit-done/workflows/` -> operational workflows
- `.planning/` -> planning and execution artifacts when a workflow creates them

## Notes

- Root `GEMINI.md` is only a compatibility pointer.
- Root `AGENTS.md` and `CLAUDE.md` are compatibility surfaces, not the primary workflow manual.
- If any older document conflicts with the files listed in Read Order, the Read Order files win.
